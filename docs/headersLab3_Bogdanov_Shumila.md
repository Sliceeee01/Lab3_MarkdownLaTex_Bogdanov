# Лабораторная работа №3. Работа с Markdown-разметкой и базовое использование LaTex в документации проекта 
## Markdown — язык разметки для форматирования текста с помощью простого синтаксиса. Это упрощённая альтернатива HTML, теги которого слишком сложные и объёмные. В отличие от HTML, разметка Markdown интегрируется прямо в текст, а вместо тегов использует простые символы, которые часто интуитивно понятны.
### IT Создание документации к проектам, особенно на платформах GitHub или GitLab. Например:
README-файлы в репозиториях, где описываются цели и функции проектов. 

Техническая документация к программам и библиотекам. 

Оформление комментариев в системах контроля версий. 

Документация API с примерами кода и запросов. 

Пользовательские руководства с пошаговыми инструкциями. 

Внутренние вики-системы команд и организаций. 

Также Markdown используется для:
Ведения заметок и организации информации в приложениях, поддерживающих разметку (Notion, Obsidian и другие).
Форматирования чек-листов и списков дел в системах отслеживания задач (Trello, Jira).

#### Markdown нужен разработчикам для форматирования текста, делая его читабельным как в исходном виде, так и в преобразованном формате, например в HTML. Цель — обеспечить простой способ форматирования, который при этом оставался бы легко читаемым даже в исходной форме без преобразования.
